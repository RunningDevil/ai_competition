# 2026年ICT软件大赛参赛指导

## 1 赛题设置

本次ICT软件大赛聚焦AI辅助研发，从真实业务场景出发，设置了两个赛道：
| 赛道   | 名称   | 题目说明                                                     |
| :----- | :----- | :----------------------------------------------------------- |
| 赛道一 | AI难题 | <br>难题赛道分两种形式：<br>1） 由赛题组广泛收集各产品线业务团队中在AI辅助研发落地业务过程中实际遇到的关键挑战和难题，赛题组统一发布，征集高价值结题思路和方案，攻克工程工程与机制瓶颈，推动关键能力创新突破；<br>2） 由选手自拟题目，可以是你或你们团队在AI辅助研发上面临的实际业务难题，选手可以自己有命题，形式符合组委会提供的模板，可以是一个Agent、SKILL、插件等。<br>选手可以选择一个或者多个难题方向，展示你的技术思路，秀出你的解题方案。 |
| 赛道二 | AI竞赛 | 竞赛赛道聚焦AI辅助研发业务实际问题，由赛题组给出命题，参赛者运用AI技术、工程能力探索切实可行的解决方案，通过方案比选与经验沉淀，形成优秀实践和可复用资产，赋能业务落地。 |

## 2 赛制说明

本次大赛的两个赛道分为不同赛制：

### 2.1 赛道一：难题赛道

难题赛道采取资格赛形式，分为两个阶段：

- 一阶段，各参赛团队通过线上答题形式参与比赛，在提交截至日期前可以多次提交作品，平台提供作品评测的客观分数。多次提交时，以最后一次提交的作品为准；各产业评委组接口人组织产业内排序TOP5进入预审。评审团组织各产品线评委组接口人拉通对进入预审的作品进行评议，由产业评委接口人负责讲解推选理由，集体评审得出36个入围作品及TOP18作品，TOP18作品进入二阶段答辩。36个入围名额分配参考如下规则：<br> 1) ICT 的产业占总名额的2/3，每个产业推荐TOP2入围，剩余名额按高于客观分数中位值的团队数量进行差额分配。各产业按照分配的名额顺序增补推荐进入决赛的团队。<br> 2) 非ICT产业占总名额的1/3，每个产业推荐TOP1，剩余名额参考上面规则进行差额分配。
- 二阶段，评审团组织入围作品提交团队进行线上答辩，根据答辩情况评审团进行主观评分，综合客观分数得出一、二、三等奖获奖团队。

### 2.2 赛道二：竞赛赛道

竞赛赛道采用淘汰赛模式，由初赛和复赛组成，具体赛制如下：

- 初赛阶段采取线上答题形式，参赛团队在答题阶段选择一道或多道竞赛题答题，平台提供作品测评的客观分数。多次提交时，以最后一次提交的作品为准。提交时段结束后，评选36支团队进入决赛，名额分配规则如下：<br> 1) ICT 的产业占总名额的2/3，每个产业推荐TOP2入围，剩余名额按高于客观分数中位值的团队数量进行差额分配。各产业按照分配的名额顺序增补推荐进入决赛的团队。<br> 2) 非ICT产业占总名额的1/3，每个产业推荐TOP1，剩余名额参考上面规则进行差额分配。
- 决赛阶段采取各研究所线下集中答题形式，进入决赛的团队在决赛日于各研究所准备的会场完成限定时间内的题目解答，并作5min作品陈述。赛后由评审团集体评议得出各获奖团队。

## 3 赛题说明

### 3.1 产生方式

本次大赛由赛题组征集各产品线CQIC推荐的出题人提交的备选题目，经集体评议选题与优化产生，题目内容与形式在正式启动答题后在参赛平台公示。
赛道二决赛题目在决赛前仅公开题目描述，答题要求、环境等信息在决赛现场公布，出题人需遵从保密要求，不得提前泄露题目相关内容。

### 3.2 赛题形式

赛题通过大赛平台公示，包含题目背景、运行&验证验证环境、解题要求。为便于选手自验证，题目会开放部分用例，参赛团队可以在自己的环境上验证用例通过性，也可以通过赛题平台提交作品进行全用例验证。
为保障平台算力合理利用，同时确保未开放用例不泄露，平台限制每日可提交验证次数不超过5次，且仅显示客观评分与用例通过比例，不显示用例失败详情。

## 4 答题要求

本次大赛答题使用线上平台提交作品，在答题截止时间前，可多次上传，以最后一次上传的作品为最终答题作品。
作品以zip压缩包形式交付，文件名以 ***{赛道ID}_{题号}_{队名}.zip*** 格式命名，难题赛道ID为01，竞赛赛道ID为02，例如：**01_01_攻坚克难队.zip**。压缩包解压后应**以同名的文件夹为根目录**，目录内格式以及关键交付件如下表：

## 5 评分方式

### 5.1 积分规则

除赛道一选手自行拟定的题目外，每道题目均包含客观分数和主观分数两部分组成。分数说明如下：
客观评分：60分，由两部分组成，由平台根据/INSTRUCTION.md加载运行结果判定
| 考核点 | 分值 | 评分说明                                                     |
| :----- | :--- | :----------------------------------------------------------- |
| 准确性 | 30分 | 取5次运行结果中，通过率最高的一次的通过率*30分。             |
| 稳定性 | 30分 | 清空上下文独立运行五次，五次运行均通过的用例数/总用例数*30分。 |

主观分数：40分，由三部分组成，由评委根据以下参考进行评分：
| 考核点 | 分值 | 评分说明                                                     |
| :----- | :--- | :----------------------------------------------------------- |
| 泛用性 | 20分 | 审视作品的实现以及用例通过情况，评估作品在同类场景中的泛化能力。对于出现根据题目的验证环境进行硬编码的情况酌情予以扣减。 |
| 规范性 | 10分 | 参考公司Skill规范/SPEC规范/MCP规范，评估作品的规范性。       |
| 性能   | 10分 | 平台提供作品的运行时长、Token消耗（包含5次的均值和最大最小值）,供评委依据同一题目其他作品的性能进行分阶排序。 |

对于选手自行拟定的题目，根据第二次大赛答疑会议，客观评分仅用于帮助评委判定自行拟定题目的作品的稳定性和准确性，不参与客观评分中位值排序，综合得分由评委根据题目设置的应用价值和难度主观评定。

### 5.2 主观分数评定方式

#### 5.2.1 产业内排序评定

根据赛制说明，不同阶段的评委分别负责该阶段的主观评分，如赛道一阶段一和赛道二初赛阶段。
由各产业评委代表负责，组织产业内评委评选，参考积分规则进行独立打分，提供排序依据，产业内集体评议完成排序推荐。通过邮件或纪要向大赛组委会正式提交。

#### 5.2.2 产业评委与评审团集体评议

赛道一阶段二由入围团队进行公开答辩，产业评委负责人与评审团成员联合在线评审，参考客观评分数据，评审成员独立打分，去除两个最高、两个最低分后，取评分均值作为最终分数，有较大差异和争议的分数由评审成员合议最终分数。
赛道一阶段二获奖团队根据最终分数排序结果评定。

#### 5.2.3 评审团评选

在赛道二决赛阶段，由评审团成员独立打分，去除两个最高、两个最低分后，取评分均值作为最终分数，有较大差异和争议的分数由评审团集体审视合议最终分数。
赛道二阶段决赛获奖团队根据最终分数排序结果评定。 不用回复我
